## 1. 产品定位

### 1.1 一句话定位

**面向同人与原创写作者的、以"主动闭环研究"替代"被动问答检索"的AI写作资料助手。**

### 1.2 核心价值主张

| 传统方式（ChatGPT/搜索引擎） | 本产品 |
|---|---|
| 用户自己拆问题、自己搜、自己整理 | Agent主动澄清需求、规划研究、提取设定、组装交付 |
| 输出散落在多轮对话中 | 一次性交付结构化设定包（SettingPackage） |
| 无校验，幻觉内容直接混入设定 | Checker校验+确定性组装，生成稳定率100% |
| 每次写作都要重新检索 | 设定包可复用、可版本管理 |

### 1.3 产品形态

- **交互界面**：对话式（Chat UI）+ 结构化输出面板
- **核心引擎**：6步LangGraph状态机驱动的Agent工作流
- **知识底座**：三层RAG（通用资料 / 写作技法 / 项目私设）
- **交付物**：SettingPackage（设定包）+ SettingCard（设定卡）

## 2. 目标用户画像

### 2.1 用户A：同人写作者（核心用户）

| 属性 | 描述 |
|---|---|
| 身份 | 二次元/影视/游戏同人创作者，活跃于LOFTER、AO3、Pixiv |
| 行为 | 开坑前需要收集原作设定、考据细节、同人圈已有私设 |
| 痛点 | ①资料分散在wiki、论坛、SNS，收集耗时 ②整理成自己能用的设定表效率低 ③担心与原作设定冲突或与其他同人撞设定 |
| 目标 | 快速拿到一份"可以放心用的、不冲突的"设定包 |

### 2.2 用户B：原创写作者（扩展用户）

| 属性 | 描述 |
|---|---|
| 身份 | 网文作者、轻小说作者、跑团GM |
| 行为 | 需要从零构建世界观、人物关系、设定体系 |
| 痛点 | ①世界观设定容易前后矛盾 ②缺少系统工具来管理设定一致性 ③研究灵感来源零散 |
| 目标 | 高效完成世界观研究，获得结构化的设定体系 |

### 2.3 用户C：写作社团/工作室（远期用户）

多作者协作场景，需要共享设定库、避免设定冲突、统一世界观。

## 3. 核心用户旅程

### 3.1 主路径："我要写一个XX同人"

```
用户输入
  │ "我想写一篇《咒术回战》的现代AU同人，五条悟×原创女主，需要完整的
  │  世界观设定和人物关系，尤其是咒术体系的现代化改编"
  ▼
Step1 任务澄清 (Planner + Clear)
  │ Agent反问3-5个澄清问题：
  │  · "现代AU是指完全去除咒术改为科技体系，还是保留咒术但设定在当代？"
  │  · "原创女主的身份定位？(同为咒术师/普通人/咒灵/天元相关)"
  │  · "需要保留哪些原作角色关系？"
  │  · "希望结局基调是HE还是BE？"
  ▼
用户确认澄清
  │
  ▼
Step2 研究规划 (Planner)
  │ Agent生成ResearchPlan JSON：
  │  · 研究维度：原作设定考据、AU改编方向、现代科技×咒术融合案例、
  │    人物性格锚点、世界观规则列表
  │  · 每个维度的检索query和优先级
  ▼
Step3 资料检索 (Researcher)
  │ 三层RAG并行检索：
  │  · L1 通用资料层 → 咒术回战wiki、角色设定
  │  · L2 写作技法层 → AU写作方法论、世界观构建技法
  │  · L3 项目私设层 → 用户历史设定包（如存在）
  ▼
Step4 设定提取 (Extractor)
  │ LLM对每批检索结果提取SettingCard：
  │  · 每张卡片 = {category, key, value, source, confidence}
  │  · 仅输出小JSON（200-500 tokens/卡）
  ▼
Step5 设定整理 (Checker)
  │ 冲突检测 → 去重 → 归类：
  │  · 示例冲突：L1说咒力来自宿傩手指，L3用户私设咒力来自基因突变
  │  · Agent标记冲突位置，请求用户仲裁
  ▼
用户仲裁冲突
  │
  ▼
Step6 设定交付 (Formatter)
  │ 确定性组装引擎：
  │  · 模板+规则拼装SettingPackage（非LLM生成）
  │  · 输出可下载的设定包和设定卡
  ▼
用户获得完整设定包 ✓
  │  · 世界观设定卡 x12
  │  · 人物设定卡 x8
  │  · 关系图谱数据 x1
  │  · 研究来源索引 x1
```

### 3.2 异常路径

- **澄清死循环保护**：最多反问5轮，超出则给出最佳猜测+标注待确认
- **检索无结果**：扩大RAG检索范围，降级到L1-only模式
- **冲突无法仲裁**：标记为"待确认"，在设定包中高亮

## 4. 功能需求列表

### 4.1 P0 — MVP核心（已实现）

| ID | 功能 | 描述 |
|---|---|---|
| F01 | 6步Agent工作流 | 完整执行任务澄清→研究规划→资料检索→设定提取→设定整理→设定交付 |

| F06 | SettingPackage导出 | 输出可下载的Markdown/JSON格式设定包 |

## 5. 非功能需求

### 5.1 性能

| 指标 | 目标值 | 测量方法 |
|---|---|---|
| 端到端工作流完成时间 | ≤ 120s（含用户交互等待） | 从任务澄清提交到设定包就绪 |
| 单次RAG检索延迟 | ≤ 3s | L1/L2/L3并行检索P95延迟 |
| 设定卡提取延迟 | ≤ 10s/批次 | 单次Extractor调用P95延迟 |
| 组装引擎渲染 | ≤ 1s | SettingPackage生成P99延迟 |
| 并发用户数 | 支持10并发（MVP） | 压力测试 |

### 5.2 质量

| 指标 | 目标值 | 当前实测值 |
|---|---|---|
| JSON解析成功率 | ≥ 90% | 90%（从40%提升） |
| 字段匹配率 | ≥ 85% | 85%（从30%提升） |
| 生成稳定率 | 100% | 100%（从50%提升） |
| 一致性检查命中率 | ≥ 70% | 约75% |
| 研究维度覆盖率 | 100%（基于ResearchPlan维度） | 100% |

### 5.3 可靠性

- **确定性保证**：组装环节零LLM参与，模板+规则保证同输入必同输出
- **降级策略**：RAG检索失败时降级为仅L1通用资料；LLM调用失败时使用缓存结果
- **幂等性**：相同输入多次执行产生相同SettingPackage（除时间戳等元数据外）

### 5.4 可扩展性

- Skill插件化：新增Skill只需注册到LangGraph状态机
- RAG层可扩展：支持添加L4、L5等自定义知识层
- 设定卡Schema可演进：支持向后兼容的字段扩展

## 6. 竞品分析

### 6.1 竞品矩阵

| 维度 | 本产品 | ChatGPT | Notion AI | Character.AI | NovelAI |
|---|---|---|---|---|---|
| **工作模式** | 主动闭环Agent | 被动问答 | 被动问答+文档 | 角色扮演 | 辅助续写 |
| **设定管理** | 结构化设定包 | 无 | 文档级 | 角色级 | 有限 |
| **研究能力** | 6步规划+RAG | 依靠用户提示 | 依靠用户输入 | 无 | 无 |
| **冲突检测** | 自动+仲裁 | 无 | 无 | 无 | 无 |
| **输出稳定性** | 100%（确定性组装） | ~50% | ~60% | ~40% | ~70% |
| **私设支持** | 三层RAG自动定位 | 需手动说明 | 需手动管理 | 有限 | 有限 |

### 6.2 差异化优势

1. **主动闭环 vs 被动问答**：用户不需要自己拆解研究步骤，Agent规划并执行完整流程
2. **确定性输出**：组装引擎使输出100%稳定，解决LLM输出的随机性问题
3. **私设冲突感知**：自动检测并定位设定冲突，这是所有竞品都不具备的能力
4. **可复用资产**：设定包不是一次性对话记录，而是可管理、可迁移的结构化资产

### 6.3 竞争风险

- **ChatGPT/Claude原生能力提升**：若ChatGPT达到同等规划与结构化输出水平，差异化缩减
- 应对策略：深耕垂直场景（同人写作的特定需求）、积累私设知识库壁垒

## 9. 风险与假设

### 9.1 核心假设

| 假设 | 验证方法 |
|---|---|
| 同人写作者愿意为结构化设定管理付费 | MVP用户访谈+付费意愿调研 |
| 6步工作流比被动问答效率提升显著 | A/B测试：本产品 vs ChatGPT完成同一任务 |
| 确定性组装引擎确实消除输出随机性 | 100次重复执行验证 |

### 9.2 已知风险

| 风险 | 等级 | 缓解措施 |
|---|---|---|
| LLM API成本随用户量线性增长 | 中 | 缓存策略+设定卡模板复用+批量调用优化 |
| 原作版权资料使用边界模糊 | 高 | 仅索引公开wiki，不存储原作原文 |
| 用户对Agent主动性接受度不足 | 低 | 澄清阶段透明化Agent意图，提供"跳过"选项 |
